🐍 Django Backend: Observability & CI/CD Platform




Профессионально настроенное backend-приложение на Django, развернутое с применением методологии Infrastructure as Code (IaC). Проект демонстрирует полную автоматизацию жизненного цикла ПО: от сборки до глубокого мониторинга и логирования.
🛠 Технологический стек

    📦 Backend: Python 3.11, Django, Gunicorn (WSGI Server).
    🐳 Containerization: Docker, Docker Compose.
    ⚙️ CI/CD: GitLab CI (архитектура на базе Templates, Includes и Extends).
    🤖 Configuration Management: Ansible (автоматизация деплоя и настройки узлов).
    📊 Monitoring: Prometheus + cAdvisor + Grafana.
    📝 Logging: Grafana Loki + Promtail.

🚀 Ключевые инженерные решения
🏗 CI/CD Архитектура

    🧩 Централизованные шаблоны: Конфигурация вынесена в .gitlab-ci-template.yml, что позволяет масштабировать CI/CD на 100+ проектов.
    🔄 Наследование (Extends): Реализовано гибкое переопределение переменных для специфических задач (Overriding).
    ⚡ Оптимизация: Проброс docker.sock для использования кэша хост-системы, что ускоряет сборку образов.

📜 Автоматизированный деплой (Ansible)

    💎 Идемпотентность: Гарантированное целевое состояние системы при каждом запуске плейбука.
    🧹 Self-healing: Механизм предварительной очистки (pre-cleanup) ошибочных директорий и конфликтующих контейнеров.
    📂 IaC: Декларативная доставка конфигураций мониторинга и управление правами доступа (0755).

📊 Наблюдаемость (Observability)

    📈 Metrics: Сбор системных метрик контейнеров (CPU, RAM, Network) через cAdvisor и Prometheus.
    🪵 Logging: Реализован стек Loki + Promtail. Сбор логов напрямую из Docker-контейнеров с фильтрацией по метаданным.
    🧱 Ротация данных: Ограничение размера логов (10m) и времени хранения метрик (7d) для стабильной работы на серверах с малым диском (10GB).

💾 Оптимизация ресурсов

    🧠 Память: Настройка Swap-файла (1GB) и жестких лимитов в docker-compose.yml для работы тяжелого стека мониторинга на 1GB RAM.
    🛡 Стабильность: Использование Gunicorn и изоляция сетей через my_bridge_net.

📋 Структура проекта

    🛠 Build Stage: Сборка Docker-образа.
    🧪 Test Stage: Unit-тестирование и линтинг кода (flake8).
    🚢 Deploy Stage: Декларативный запуск всей инфраструктуры через Ansible.
    📊 Monitoring Stage: Визуализация состояния системы в Grafana (Порт 3000).

🔧 Быстрый запуск

    Клонировать репозиторий:
    bash

    git clone https://gitlab.com

    Используйте код с осторожностью.
    Развернуть весь стек:
    bash

    docker-compose up -d --build

    Используйте код с осторожностью.
